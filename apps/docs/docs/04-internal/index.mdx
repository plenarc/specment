# 内部設計

:::tip
1. 何故その言語、フレームワークを選定したかを書いておく
1. 実装が済めば(リリースされれば)詳細なことはソースに書かれているはず
    1. なので仕様には「なぜそのコードに至ったか？」など実装ではかかれないことを明記することに注力する
:::
